Software Life Cycle: Development phase Definitions
Requirements analysis Specifies what services the proposed system will provide, to identify any conditions on those services, and to define how the outside world will interact with the system.
Design Creating a plan for the construction of the proposed system.
Implementation The actual writing of programs, creation of data files, and development of databases.
Testing One segment in the overall struggle for quality assurance, the direct definition is testing the essentially equated process of debugging programs and confirming that the product was compatiblr with the software requirements specification. Today we now know that one test can not be overall correct and that it should be incorporated to the other three steps.

Data Relationships Definitions
One-to-One The association between patients and occupied private rooms in that each patient is associated with only one room and each room is associated with only one patient.
One-to-Many The association between physicians and patients in that one physician is associated with many patients and each patient is associated with one physician.
Many-to-Many If you include consulting physicians in the physician-patient relationship. Then many patients could be associated with many physicians and vice versa.
Terms Definitions
Design Pattern A predeveloped model for solving a recurring problem in software design.
Software Quality Assurance (SQA) The overseeing and enforcing the quality control systems adopted by the organization.
Software Testing Definitions
Glass Box Testing The software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
Black Box Testing Tests that do not rely on knowledge of the softwares interior composition. Performed from the users point of view.
Beta Testing This falls within black-box testing, which a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and realeased to the market.
Alpha Testing This is very similar to beta testing.
Documentation Definitions
User Documentation This is used to explain the features of the software and describe how to use them.
System Documentation Used to describe the softwares internal composition so that they software can be maintained later in its life cycle.
Technical Documentation Used to describe how a software system should be installed and serviced installing updates, and reporting problems back to the softwares developer.
